# 文档协作工作流

完整手册：`read_file(".reasonix/skills/anthropic/doc-coauthoring/SKILL.md")`

## 三阶段流程

### 阶段 1：上下文收集
1. 问元问题：文档类型？受众？期望效果？模板？约束？
2. 鼓励信息倾泻（背景、讨论、替代方案、组织上下文）
3. 5-10 个澄清问题

### 阶段 2：精炼与结构
逐节构建：
1. 澄清问题 → 2. 头脑风暴 5-20 选项 → 3. 用户筛选 → 4. 查漏 → 5. 起草 → 6. 迭代精炼

### 阶段 3：读者测试
用全新视角测试文档是否能被理解。

## 指南

- 直接、程序化，不过度推销
- 用户可跳过或自定义流程
- 质量优先于速度
